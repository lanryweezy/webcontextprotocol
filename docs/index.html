Web Context Protocol (WCP)

A proposed standard for making web applications understandable to AI agents.

Introduction

The Web Context Protocol (WCP) is a simple HTML attribute, data-wcp, designed to provide clear and explicit context to AI agents. It helps developers control how AI interacts with their web pages, ensuring predictable and safe behavior.

Syntax

The data-wcp attribute contains semicolon-separated key-value pairs:

key:value; key2:value2;

Example

<button data-wcp="action:add; object:cart; group-id:product-123">Add to Cart</button>

Core Schema (v0.2)

The following keys form the core of the WCP specification.

Interaction Keys

action

Describes the primary purpose of an interactive element.

Value Description
addAdds an item to a collection (e.g., cart).
purchaseInitiates a checkout process.
searchPerforms a search.
submitSubmits a form.
deleteDeletes an item.
cancelCancels an action.
navigateNavigates to a new page or view.
toggleToggles the state of an element.

effect

Describes the expected outcome of an interaction.

Value Description
asyncAction is performed asynchronously (no page reload).
navigateAction causes navigation to a new URL.
modalAction opens a modal dialog.
downloadAction initiates a file download.

Form Input Keys

purpose

Describes the kind of information expected in a form field.

<input type="email" data-wcp="purpose:email">

Data Extraction Keys

value

Identifies an element that contains a specific data point.

<span data-wcp="value:price; currency:USD">19.99</span>

Context and Relationship Keys

state

Describes the current state of a stateful element (e.g., used with action:toggle).

<button data-wcp="action:toggle; state:on">Notifications</button>

group-id

Associates multiple elements that belong to the same conceptual group.

<h2 data-wcp="value:product-name; group-id:product-456">Product Name</h2>
<button data-wcp="action:add; group-id:product-456">Add to Cart</button>
